# motif Philosophy

- 静态优先：尽量在构建期完成扫描与生成。
- 双 preset 明确：当前只做 `f-` 与 `m-` 两套内建 preset。
- 预设优先：不是让使用者自己拼风格，而是直接拿到 Win11 向或 Google 向的预设结果。
- 框架无关：核心产物是 CSS。
- 能力分层：扫描、解析、规则、token、生成、输出分开。
- 最小可行：先完成闭环，再扩能力。
- 简洁优先：宁可少做，也不要把系统做成难以理解的怪物。
- 清晰优先：不要让使用者和维护者去猜规则如何生效。
- 稳扎稳打：每一步都要可验证，不靠补丁堆进度感。
